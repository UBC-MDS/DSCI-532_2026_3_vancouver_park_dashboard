# Contributing to Vancouver Park Dashboard Project

The following are the steps to contribute to our project.

## General guidelines

Before writing a pull request, please open an issue first and wait for our project administrators to approve your ideas. If you **find a bug** in our code, please reach out to us by an issue and/or emailing us. In the discussion, please demonstrate how the bug happens using a minimal reproducible example. To **submit feedback**, please let us know also by an issue. We appreciate thoughtful ideas, but this is a volunteer-driven project and everybody has limited time.

## Pull request

* Right now we only accept PR's created by project members.

## Code style

* Your coding style needs to follow [PEP 8](https://pep8.org/) Python style guide.

* Using easy-to-understand variable names and keeping one function per task are appreciated.

* Please document docstrings under all functions and classes definitions using [Google style](https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings).

* For Jupyter notebooks:
  * Please appropriately use markdown cells to literate the code
  * Code cells should be well formatted and commented
  * Visualizations should come with proper titles and human-readable labels

## Code of Conduct

Please note that the project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By contributing to this project you agree to abide by its terms.
